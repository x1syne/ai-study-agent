## Описание проекта

AI Study Agent — это веб-приложение для самообразования, которое использует передовые технологии искусственного интеллекта для создания персонализированных учебных курсов. Система автоматически генерирует теоретический материал, практические задания и адаптируется под уровень знаний каждого пользователя.

### Ключевые особенности

- Автоматическая генерация курсов по любой теме с помощью LLM
- Адаптивная система обучения, подстраивающаяся под пользователя
- Интервальное повторение на основе алгоритма SM-2
- Геймификация учебного процесса (XP, уровни, достижения)
- AI-репетитор для ответов на вопросы в реальном времени
- Визуальный граф знаний с зависимостями между темами

## Инновационные технологии

### 1. Agentic AI (Агентный искусственный интеллект)

Проект реализует концепцию **Agentic AI** — одного из главных технологических трендов 2025 года по версии Gartner. В отличие от простых чат-ботов, агентная система способна:

- **Автономно принимать решения** о структуре курса
- **Анализировать контекст** и адаптировать материал
- **Планировать последовательность** обучения
- **Самостоятельно выбирать** методы подачи информации

#### Трёхэтапная архитектура AI-агента

```
┌─────────────────────────────────────────────────────────────┐
│  ЭТАП 1: AI-АНАЛИТИК                                        │
│  Анализирует тему и определяет:                             │
│  - Природу темы (концептуальная/процедурная/фактическая)   │
│  - Оптимальный метод обучения                               │
│  - Формат контента                                          │
│  - Связи с реальными применениями                           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  ЭТАП 2: ДИНАМИЧЕСКИЙ КОНСТРУКТОР                           │
│  Строит уникальную структуру курса:                         │
│  - Адаптивное введение под тип темы                         │
│  - Секции зависят от сложности (1-10)                       │
│  - Практика зависит от метода обучения                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  ЭТАП 3: УМНЫЙ ГЕНЕРАТОР                                    │
│  Создаёт контент секция за секцией:                         │
│  - Использует внешние источники (arXiv, Open Library)       │
│  - Адаптирует стиль под тип темы                            │
│  - Генерирует интерактивные элементы                        │
└─────────────────────────────────────────────────────────────┘
```

#### Классификация тем (TopicNature)

AI автоматически определяет природу темы для выбора оптимальной стратегии обучения:

| Тип | Описание | Пример | Метод обучения |
|-----|----------|--------|----------------|
| `conceptual` | Абстрактные идеи и теории | Квантовая физика, философия | Аналогии → Формализация → Применение |
| `procedural` | Пошаговые процессы | Программирование, рецепты | Демонстрация → Практика → Проект |
| `factual` | Факты, даты, данные | История, география | Контекст → Хронология → Связи |
| `skill` | Практические умения | Музыка, спорт | Наблюдение → Имитация → Создание |
| `creative` | Творческие навыки | Рисование, писательство | Вдохновение → Техники → Эксперименты |



### 2. Large Language Models (LLM) и Prompt Engineering

Система использует **Groq API** с моделями семейства LLaMA для генерации контента. Реализована каскадная архитектура с автоматическим переключением между моделями:

```
Основная модель: LLaMA 3.3-70B Versatile
        ↓ (при rate limit)
Fallback 1: LLaMA 3.1-70B Versatile
        ↓ (при ошибке)
Fallback 2: LLaMA 3.1-8B Instant
        ↓ (при ошибке)
Fallback 3: Mixtral 8x7B-32768
```

#### Система промптов

Разработана специализированная система промптов для разных задач:

| Промпт | Назначение | Особенности |
|--------|------------|-------------|
| `graphGeneration` | Генерация структуры курса | Создаёт 8-15 тем с зависимостями |
| `theory` | Генерация теории | Минимум 3000 слов, интерактивные блоки |
| `taskGeneration` | Создание заданий | 6 типов, прогрессия сложности |
| `codeReview` | Проверка кода | Семантический анализ через AI |
| `chat` | AI-репетитор | Generative UI компоненты |

#### Generative UI

AI-чат способен генерировать интерактивные UI-компоненты прямо в ответах:

- **quiz** — интерактивные тесты с проверкой
- **comparison** — таблицы сравнения
- **code** — примеры кода с подсветкой
- **flowchart** — блок-схемы алгоритмов
- **steps** — пошаговые инструкции
- **info** — информационные карточки

### 3. Retrieval-Augmented Generation (RAG)

Для повышения качества генерируемого контента используется технология **RAG** — обогащение контекста из внешних источников:

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   arXiv API     │     │ Open Library    │     │   RAG Search    │
│ (научные статьи)│     │    (книги)      │     │ (локальный поиск)│
└────────┬────────┘     └────────┬────────┘     └────────┬────────┘
         │                       │                       │
         └───────────────────────┼───────────────────────┘
                                 │
                                 ▼
                    ┌─────────────────────────┐
                    │   Объединённый контекст │
                    │   для LLM генерации     │
                    └─────────────────────────┘
```

Это позволяет:
- Использовать актуальные научные данные
- Ссылаться на авторитетные источники
- Генерировать более точный и глубокий контент

### 4. Adaptive Learning (Адаптивное обучение)

Система анализирует результаты пользователя и автоматически адаптирует процесс обучения:

#### Механизм адаптации

1. **Анализ ответов** — отслеживание правильных/неправильных ответов
2. **Расчёт mastery level** — уровень владения темой (0-100%)
3. **Рекомендации** — при низком проценте предлагается повторить теорию
4. **Динамическая сложность** — задания сортируются от простых к сложным

```
Результат < 40%  →  Рекомендация повторить теорию
Результат 40-70% →  Дополнительная практика
Результат > 70%  →  Переход к следующей теме
```

#### Прогрессия сложности заданий

Все задания автоматически сортируются по уровню сложности:

| Уровень | Количество | Тип задач |
|---------|------------|-----------|
| Easy | 5 заданий | Базовое понимание, определения |
| Medium | 6 заданий | Применение знаний, расчёты |
| Hard | 4 задания | Анализ, нестандартные ситуации |

### 5. Spaced Repetition (Интервальное повторение)

Реализован алгоритм **SM-2** (SuperMemo 2) для оптимального запоминания материала:

#### Формула расчёта интервала

```
EF' = EF + (0.1 - (5 - q) × (0.08 + (5 - q) × 0.02))

где:
  EF  — фактор лёгкости (начальное значение 2.5)
  q   — качество ответа (0-5)
  EF' — новый фактор лёгкости
```

#### Расчёт следующего интервала

```
Если repetitions = 0: interval = 1 день
Если repetitions = 1: interval = 6 дней
Если repetitions > 1: interval = interval × EF
```

#### Преимущества SM-2

- Научно обоснованный метод (исследования Piotr Wozniak)
- Оптимизация времени обучения
- Долгосрочное запоминание (месяцы и годы)
- Персонализация под каждого пользователя



### 6. Gamification (Геймификация)

Система мотивации на основе игровых механик:

#### XP (Experience Points)

| Действие | XP |
|----------|-----|
| Правильный ответ (easy) | 10 XP |
| Правильный ответ (medium) | 25 XP |
| Правильный ответ (hard) | 50 XP |
| Завершение темы | 100 XP |
| Ежедневный вызов | 50 XP |

#### Система уровней

```
Level = floor(sqrt(totalXP / 100))

Примеры:
  100 XP   → Level 1
  400 XP   → Level 2
  2500 XP  → Level 5
  10000 XP → Level 10
```

#### Ранги пользователей

| Ранг | Требование XP | Цвет |
|------|---------------|------|
| Новичок | 0+ | Серый |
| Ученик | 500+ | Зелёный |
| Эксперт | 2000+ | Синий |
| Мастер | 5000+ | Фиолетовый |
| Легенда | 10000+ | Золотой |

#### Достижения (23 типа)

- **Прогресс**: Первый урок, Первая тема, Первый курс
- **Streak**: 3, 7, 30, 100 дней подряд
- **Задания**: 10, 50, 100 выполненных
- **Карточки**: 10, 50, 100 повторений
- **Особые**: Идеальный квиз, Ночная сова, Ранняя пташка

### 7. Мультимодальные задания

Система поддерживает 6 типов практических заданий:

| Тип | Описание | Проверка |
|-----|----------|----------|
| `single` | Один правильный ответ | Сравнение индексов |
| `multiple` | Несколько правильных | Проверка множества |
| `text` | Текстовый ввод | Нормализация + Levenshtein |
| `number` | Числовой ответ | Проверка с погрешностью |
| `matching` | Соответствия (Drag&Drop) | Проверка пар |
| `code` | Написание кода | AI-проверка через LLM |

#### Алгоритм проверки текстовых ответов

```
1. Нормализация: toLowerCase() + trim() + удаление пунктуации
2. Расчёт расстояния Левенштейна
3. Допуск: до 20% от длины правильного ответа
```

#### AI-проверка кода

Код пользователя отправляется в LLM для семантической проверки:
- Анализ логики решения
- Проверка тест-кейсов
- Генерация обратной связи

## Установка и запуск

### Требования

- Node.js 18+
- PostgreSQL (или Supabase)
- Groq API ключ

### 1. Клонирование репозитория

```bash
git clone https://github.com/x1syne/ai-study-agent.git
cd ai-study-agent
npm install
```

### 2. Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```env
# База данных
DATABASE_URL="postgresql://user:password@host:5432/database"

# Supabase
NEXT_PUBLIC_SUPABASE_URL="https://xxx.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your_anon_key"

# Groq AI
GROQ_API_KEY="gsk_xxx"

# Приложение
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# Опционально
USE_AI_AGENT="true"
```

### 3. Инициализация базы данных

```bash
# Генерация Prisma клиента
npm run db:generate

# Применение схемы
npm run db:push

# Заполнение начальными данными (опционально)
npm run db:seed
```

### 4. Запуск

```bash
# Режим разработки
npm run dev

# Продакшн сборка
npm run build
npm start
```

Приложение будет доступно по адресу: http://localhost:3000

## Получение API ключей

### Groq (бесплатно)

1. Зарегистрируйтесь на [console.groq.com](https://console.groq.com)
2. Создайте API ключ в разделе API Keys
3. Лимиты: 30 запросов/мин, 14,400 запросов/день

### Supabase (бесплатно)

1. Зарегистрируйтесь на [supabase.com](https://supabase.com)
2. Создайте новый проект
3. Скопируйте URL и ключи из Settings → API
4. Скопируйте DATABASE_URL из Settings → Database
