## Описание проекта

AI Study Agent — это веб-приложение для самообразования, которое использует передовые технологии искусственного интеллекта для создания персонализированных учебных курсов. Система автоматически генерирует теоретический материал, практические задания и адаптируется под уровень знаний каждого пользователя.

### Ключевые особенности

- Автоматическая генерация курсов по любой теме с помощью LLM
- Адаптивная система обучения, подстраивающаяся под пользователя
- Интервальное повторение на основе алгоритма SM-2
- Геймификация учебного процесса (XP, уровни, достижения)
- AI-репетитор для ответов на вопросы в реальном времени
- Визуальный граф знаний с зависимостями между темами

## Инновационные технологии

### 1. Agentic AI (Агентный искусственный интеллект)

Проект реализует концепцию **Agentic AI** — одного из главных технологических трендов 2025 года по версии Gartner. В отличие от простых чат-ботов, агентная система способна:

- **Автономно принимать решения** о структуре курса
- **Анализировать контекст** и адаптировать материал
- **Планировать последовательность** обучения
- **Самостоятельно выбирать** методы подачи информации

#### Трёхэтапная архитектура AI-агента

```
┌─────────────────────────────────────────────────────────────┐

│  Анализирует тему и определяет:                             │
│  - Природу темы (концептуальная/процедурная/фактическая)   │
│  - Оптимальный метод обучения                               │
│  - Формат контента                                          │
│  - Связи с реальными применениями                           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  ЭТАП 2: ДИНАМИЧЕСКИЙ КОНСТРУКТОР                           │
│  Строит уникальную структуру курса:                         │
│  - Адаптивное введение под тип темы                         │
│  - Секции зависят от сложности (1-10)                       │
```

#### Классификация тем (TopicNature)

AI автоматически определяет природу темы для выбора оптимальной стратегии обучения:

| Тип | Описание | Пример | Метод обучения |
|-----|----------|--------|----------------|
| `conceptual` | Абстрактные идеи и теории | Квантовая физика, философия | Аналогии → Формализация → Применение |
| `procedural` | Пошаговые процессы | Программирование, рецепты | Демонстрация → Практика → Проект |
| `factual` | Факты, даты, данные | История, география | Контекст → Хронология → Связи |
| `skill` | Практические умения | Музыка, спорт | Наблюдение → Имитация → Создание |
| `creative` | Творческие навыки | Рисование, писательство | Вдохновение → Техники → Эксперименты |



### 2. Large Language Models (LLM) и Prompt Engineering

Система использует **Groq API** с моделями семейства LLaMA для генерации контента. Реализована каскадная архитектура с автоматическим переключением между моделями:

```
Основная модель: LLaMA 3.3-70B Versatile
        ↓ (при rate limit)
Fallback 1: LLaMA 3.1-70B Versatile
        ↓ (при ошибке)
Fallback 2: LLaMA 3.1-8B Instant
        ↓ (при ошибке)
Fallback 3: Mixtral 8x7B-32768
```

#### Система промптов

Разработана специализированная система промптов для разных задач:

| Промпт | Назначение | Особенности |
|--------|------------|-------------|
| `graphGeneration` | Генерация структуры курса | Создаёт 8-15 тем с зависимостями |
| `theory` | Генерация теории | Минимум 3000 слов, интерактивные блоки |
| `taskGeneration` | Создание заданий | 6 типов, прогрессия сложности |
| `codeReview` | Проверка кода | Семантический анализ через AI |
| `chat` | AI-репетитор | Generative UI компоненты |

#### Generative UI

AI-чат способен генерировать интерактивные UI-компоненты прямо в ответах:

- **quiz** — интерактивные тесты с проверкой
- **comparison** — таблицы сравнения
- **code** — примеры кода с подсветкой
- **flowchart** — блок-схемы алгоритмов
- **steps** — пошаговые инструкции
- **info** — информационные карточки

### 3. Retrieval-Augmented Generation (RAG)

Для повышения качества генерируемого контента используется технология **RAG** — обогащение контекста из внешних источников:

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   arXiv API     │     │ Open Library    │     │   RAG Search    │
│ (научные статьи)│     │    (книги)      │     │ (локальный поиск)│
└────────┬────────┘     └────────┬────────┘     └────────┬────────┘
         │                       │                       │
         └───────────────────────┼───────────────────────┘
                                 │
                                 ▼
                    ┌─────────────────────────┐
                    │   Объединённый контекст │
                    │   для LLM генерации     │
                    └─────────────────────────┘
```

Это позволяет:
- Использовать актуальные научные данные
- Ссылаться на авторитетные источники
- Генерировать более точный и глубокий контент

### 4. Adaptive Learning (Адаптивное обучение)

Система анализирует результаты пользователя и автоматически адаптирует процесс обучения:

#### Механизм адаптации

1. **Анализ ответов** — отслеживание правильных/неправильных ответов
2. **Расчёт mastery level** — уровень владения темой (0-100%)
3. **Рекомендации** — при низком проценте предлагается повторить теорию
4. **Динамическая сложность** — задания сортируются от простых к сложным

```
Результат < 40%  →  Рекомендация повторить теорию
Результат 40-70% →  Дополнительная практика
Результат > 70%  →  Переход к следующей теме
```

#### Прогрессия сложности заданий

Все задания автоматически сортируются по уровню сложности:

| Уровень | Количество | Тип задач |
|---------|------------|-----------|
| Easy | 5 заданий | Базовое понимание, определения |
| Medium | 6 заданий | Применение знаний, расчёты |
| Hard | 4 задания | Анализ, нестандартные ситуации |

### 5. Spaced Repetition (Интервальное повторение)

Реализован алгоритм **SM-2** (SuperMemo 2) для оптимального запоминания материала:

#### Формула расчёта интервала

```
EF' = EF + (0.1 - (5 - q) × (0.08 + (5 - q) × 0.02))

где:
  EF  — фактор лёгкости (начальное значение 2.5)
  q   — качество ответа (0-5)
  EF' — новый фактор лёгкости
```

#### Расчёт следующего интервала

```
Если repetitions = 0: interval = 1 день
Если repetitions = 1: interval = 6 дней
Если repetitions > 1: interval = interval × EF
```

#### Преимущества SM-2

- Научно обоснованный метод (исследования Piotr Wozniak)
- Оптимизация времени обучения
- Долгосрочное запоминание (месяцы и годы)
- Персонализация под каждого пользователя



### 6. Gamification (Геймификация)

Система мотивации на основе игровых механик:

#### XP (Experience Points)

| Действие | XP |
|----------|-----|
| Правильный ответ (easy) | 10 XP |
| Правильный ответ (medium) | 25 XP |
| Правильный ответ (hard) | 50 XP |
| Завершение темы | 100 XP |
| Ежедневный вызов | 50 XP |

#### Система уровней

```
Level = floor(sqrt(totalXP / 100))

Примеры:
  100 XP   → Level 1
  400 XP   → Level 2
  2500 XP  → Level 5
  10000 XP → Level 10
```

#### Ранги пользователей

| Ранг | Требование XP | Цвет |
|------|---------------|------|
| Новичок | 0+ | Серый |
| Ученик | 500+ | Зелёный |
| Эксперт | 2000+ | Синий |
| Мастер | 5000+ | Фиолетовый |
| Легенда | 10000+ | Золотой |

#### Достижения (23 типа)

- **Прогресс**: Первый урок, Первая тема, Первый курс
- **Streak**: 3, 7, 30, 100 дней подряд
- **Задания**: 10, 50, 100 выполненных
- **Карточки**: 10, 50, 100 повторений
- **Особые**: Идеальный квиз, Ночная сова, Ранняя пташка

### 7. Мультимодальные задания

Система поддерживает 6 типов практических заданий:

| Тип | Описание | Проверка |
|-----|----------|----------|
| `single` | Один правильный ответ | Сравнение индексов |
| `multiple` | Несколько правильных | Проверка множества |
| `text` | Текстовый ввод | Нормализация + Levenshtein |
| `number` | Числовой ответ | Проверка с погрешностью |
| `matching` | Соответствия (Drag&Drop) | Проверка пар |
| `code` | Написание кода | AI-проверка через LLM |

#### Алгоритм проверки текстовых ответов

```
1. Нормализация: toLowerCase() + trim() + удаление пунктуации
2. Расчёт расстояния Левенштейна
3. Допуск: до 20% от длины правильного ответа
```

#### AI-проверка кода

Код пользователя отправляется в LLM для семантической проверки:
- Анализ логики решения
- Проверка тест-кейсов
- Генерация обратной связи

### 8. Model Context Protocol (MCP) Integration

Система интегрирована с **Model Context Protocol (MCP)** — открытым стандартом для подключения языковых моделей к внешним инструментам и данным. Это превращает AI-агента из простого генератора текста в полноценного помощника, способного выполнять реальные действия.

#### Возможности MCP

```
┌─────────────────────────────────────────────────────────────┐
│                      AI Study Agent                          │
│                                                              │
│  ┌──────────────┐      ┌──────────────┐                    │
│  │   AI Chat    │      │ Theory Gen   │                    │
│  │              │      │              │                    │
│  │  - Memory    │      │ - State      │                    │
│  │  - Context   │      │   Machine    │                    │
│  │  - MCP Tools │      │ - Retry      │                    │
│  └──────┬───────┘      └──────┬───────┘                    │
│         │                     │                             │
│         └──────────┬──────────┘                             │
│                    │                                        │
│         ┌──────────▼──────────┐                            │
│         │   MCP Client        │                            │
│         │                     │                            │
│         │  - Server Manager   │                            │
│         │  - Tool Router      │                            │
│         │  - Error Handler    │                            │
│         └──────────┬──────────┘                            │
│                    │                                        │
└────────────────────┼────────────────────────────────────────┘
                     │
         ┌───────────┴───────────┐
         │                       │
    ┌────▼─────┐          ┌─────▼────┐
    │Filesystem│          │  Brave   │
    │  Server  │          │  Search  │
    └──────────┘          └──────────┘
```

#### MCP Серверы

| Сервер | Назначение | Инструменты |
|--------|------------|-------------|
| **Filesystem** | Работа с файлами | `read_file`, `write_file`, `list_directory`, `create_directory`, `delete_file` |
| **Brave Search** | Веб-поиск | `brave_web_search` |

#### Примеры использования

**Сохранение кода:**
```
Студент: "Сохрани этот код в файл example.js"
AI: Сохраняет файл через Filesystem Server
    Возвращает ссылку для скачивания
```

**Поиск информации:**
```
Студент: "Какие новые фичи в React 19?"
AI: Выполняет поиск через Brave Search
    Возвращает актуальные результаты с источниками
```

#### Безопасность MCP

- **Path Validation**: Все пути файлов валидируются на directory traversal
- **User Isolation**: Файлы каждого пользователя в отдельной директории
- **API Keys**: Не передаются на клиент, только на сервере
- **Rate Limiting**: Кэширование поиска, retry с backoff
- **Input Sanitization**: Имена файлов очищаются от опасных символов

#### Дополнительные возможности

- **Contextual Memory**: AI помнит контекст разговора (до 10 сообщений)
- **Retry Mechanism**: Автоматические повторы при сбоях (до 3 раз)
- **State Machine**: Управление состоянием генерации контента
- **Task Classification**: AI-классификация сложности заданий

## Установка и запуск

### Требования

- Node.js 18+
- PostgreSQL (или Supabase)
- Groq API ключ

### 1. Клонирование репозитория

```bash
git clone https://github.com/x1syne/ai-study-agent.git
cd ai-study-agent
npm install
```

### 2. Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```env
# База данных
DATABASE_URL="postgresql://user:password@host:5432/database"

# Supabase
NEXT_PUBLIC_SUPABASE_URL="https://xxx.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="your_anon_key"

# Groq AI (обязательно)
GROQ_API_KEY="gsk_xxx"

# MCP Integration (опционально)
MCP_ENABLED=true
BRAVE_API_KEY="BSA_xxx"  # Для веб-поиска
MCP_FILESYSTEM_PATH="./user-files"  # Путь для файлов пользователей

# Приложение
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# Опционально
USE_AI_AGENT="true"
SERPER_API_KEY="your_serper_api_key"  # Для RAG поиска
```

#### Обязательные переменные

- `DATABASE_URL` — подключение к PostgreSQL
- `GROQ_API_KEY` — ключ для Groq AI (получить на [console.groq.com](https://console.groq.com))
- `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY` — для Supabase

#### MCP переменные (опционально)

- `MCP_ENABLED` — включить/выключить MCP интеграцию (по умолчанию `true`)
- `BRAVE_API_KEY` — ключ для Brave Search API (получить на [brave.com/search/api](https://brave.com/search/api/))
  - Бесплатный тариф: 2000 запросов/месяц
  - Используется для веб-поиска в AI-чате
- `MCP_FILESYSTEM_PATH` — путь для хранения файлов пользователей (по умолчанию `./user-files`)

**Примечание:** Если `BRAVE_API_KEY` не указан, веб-поиск будет недоступен, но остальные функции будут работать.

### 3. Инициализация базы данных

```bash
# Генерация Prisma клиента
npm run db:generate

# Применение схемы
npm run db:push

# Заполнение начальными данными (опционально)
npm run db:seed
```

### 4. Запуск

```bash
# Режим разработки
npm run dev

# Продакшн сборка
npm run build
npm start
```

Приложение будет доступно по адресу: http://localhost:3000

## MCP Configuration

### Настройка MCP серверов

MCP серверы настраиваются в файле `.kiro/settings/mcp.json`. По умолчанию настроены два сервера:

#### 1. Filesystem Server

Позволяет AI сохранять и читать файлы:

```json
{
  "filesystem": {
    "name": "filesystem",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "./user-files"],
    "disabled": false,
    "autoApprove": ["read_file", "write_file", "list_directory"]
  }
}
```

**Возможности:**
- Сохранение кода, заметок, примеров
- Чтение файлов пользователя
- Список файлов в директории
- Создание и удаление файлов

**Безопасность:**
- Все файлы изолированы в user-specific директориях
- Валидация путей для предотвращения directory traversal
- Санитизация имён файлов

#### 2. Brave Search Server

Позволяет AI искать информацию в интернете:

```json
{
  "brave-search": {
    "name": "brave-search",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-brave-search"],
    "env": {
      "BRAVE_API_KEY": "${BRAVE_API_KEY}"
    },
    "disabled": false,
    "autoApprove": ["brave_web_search"]
  }
}
```

**Возможности:**
- Поиск актуальной информации
- Поиск документации библиотек
- Поиск примеров кода
- Поиск научных статей

**Оптимизация:**
- Результаты кэшируются на 1 час
- Ограничение до 5 результатов
- Автоматическое определение необходимости поиска

### Проверка статуса MCP

После запуска приложения проверьте логи:

```bash
npm run dev

# Должны увидеть:
# [MCP] Initialization complete
# [MCP] Filesystem server: running
# [MCP] Brave-search server: running (или disabled если нет API ключа)
```

Также можно проверить статус в настройках приложения (Settings → MCP Servers).

### Отключение MCP

Если хотите отключить MCP интеграцию:

```env
# В .env файле

```

Или отключите конкретный сервер в `mcp.json`:

```json
{

  }
```

## Troubleshooting (Решение проблем)

### MCP Integration Issues

#### Проблема: MCP серверы не запускаются

**Симптомы:**
- В логах нет сообщения `[MCP] Initialization complete`
- Ошибки при попытке сохранить файл или выполнить поиск

**Решение:**
1. Проверьте, что `MCP_ENABLED=true` в `.env`
2. Убедитесь, что `npx` доступен (входит в Node.js)
3. Проверьте права доступа к директории `./user-files`
4. Проверьте логи на наличие конкретных ошибок

```bash
# Создайте директорию вручную если нужно
mkdir user-files
chmod 755 user-files
```

#### Проблема: Brave Search не работает

**Симптомы:**
- Поиск не выполняется в AI-чате
- В логах: `BRAVE_API_KEY not configured`

**Решение:**
1. Получите API ключ на [brave.com/search/api](https://brave.com/search/api/)
2. Добавьте в `.env`: `BRAVE_API_KEY=BSA_xxx`
3. Перезапустите приложение

**Примечание:** Без Brave API ключа приложение работает нормально, просто без веб-поиска.

#### Проблема: Файлы не сохраняются

**Симптомы:**
- Ошибка при попытке сохранить файл через AI-чат
- `Error saving file` в логах

**Решение:**
1. Проверьте права доступа к `MCP_FILESYSTEM_PATH` (по умолчанию `./user-files`)
2. Убедитесь, что путь существует и доступен для записи
3. Проверьте, что имя файла валидное (без `..`, `/`, и других опасных символов)

```bash
# Проверка прав
ls -la user-files

# Если нужно, исправьте права
chmod -R 755 user-files
```

#### Проблема: Rate limit ошибки

**Симптомы:**
- Ошибка 429 от Brave Search API
- Слишком много запросов

**Решение:**
- Результаты поиска кэшируются на 1 час автоматически
- Бесплатный тариф: 2000 запросов/месяц
- Рассмотрите платный тариф если нужно больше запросов

### AI Generation Issues

#### Проблема: AI не отвечает или возвращает ошибку

**Симптомы:**
- `AI временно недоступен`
- Ошибки в логах от Groq API

**Решение:**
1. Проверьте `GROQ_API_KEY` в `.env`
2. Убедитесь, что ключ действителен на [console.groq.com](https://console.groq.com)
3. Проверьте лимиты: 30 запросов/мин, 14,400 запросов/день
4. Подождите несколько минут если превышен rate limit

#### Проблема: Генерация курса зависает

**Симптомы:**
- Создание курса не завершается
- Spinner крутится бесконечно

**Решение:**
1. Проверьте логи сервера на наличие ошибок
2. Убедитесь, что `USE_AI_AGENT=true` в `.env`
3. Попробуйте более простую тему
4. Проверьте подключение к интернету (для arXiv/OpenLibrary)

### Database Issues

**Симптомы:**
- `Error connecting to database`

**Решение:**
