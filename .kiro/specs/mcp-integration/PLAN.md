## Что такое MCP?

**Model Context Protocol (MCP)** - это "USB-C для ИИ", универсальный стандарт для подключения языковых моделей к внешним инструментам и данным.

### Отличие от RAG:

| Аспект | RAG | MCP |
|--------|-----|-----|
| Цель | Улучшение знаний | Подключение к данным И действиям |
| Функция | Извлечение информации | Выполнение действий + извлечение |
| Стандартизация | Концепция | Открытый стандарт |
| Пример | "Что в политике возвратов?" | "Выполни SQL-запрос и обнови дашборд" |

### Что нужно улучшить:

❌ **Линейная генерация секций** - нет retry при ошибках отдельных секций
❌ **Чат не помнит контекст** - каждое сообщение независимо
❌ **Нет веб-поиска** - AI не может искать актуальную информацию
❌ **Нет работы с файлами** - нельзя сохранить примеры кода
❌ **Нет валидации конфигурации** - ошибки обнаруживаются поздно
❌ **Классификация сложности** - не всегда точная

## Что мы добавим

### 1. MCP Filesystem Server
**Зачем:** Сохранение примеров кода, конспектов, заметок

**Как работает:**
```
Пользователь: "Сохрани этот код в файл example.js"
    ↓
AI Chat определяет намерение
    ↓
MCP Filesystem создаёт файл в user-files/[userId]/example.js
    ↓
Возвращает ссылку для скачивания
```

**Безопасность:**
- Валидация путей (защита от directory traversal)
- Изоляция файлов по пользователям
- Поддержка только текстовых форматов

### 2. MCP Brave Search Server
**Зачем:** Поиск актуальной информации в интернете

**Как работает:**
```
Пользователь: "Какие новые фичи в React 19?"
    ↓
AI Chat определяет что нужен поиск
    ↓
MCP Brave Search ищет в интернете
    ↓
Возвращает топ-5 результатов с источниками
    ↓
AI формирует ответ на основе найденного
```

**Оптимизация:**
- Кэширование результатов на 1 час
- Лимит 5 результатов
- Форматирование с источниками

### 3. Retry Mechanism
**Зачем:** Надёжность генерации контента

**Как работает:**
```
Генерация секции
    ↓
Ошибка (timeout, rate limit, etc.)
    ↓
Retry 1 (задержка 1s)
    ↓
Retry 2 (задержка 2s)
    ↓
Retry 3 (задержка 4s)
    ↓
Fallback контент
```

**Особенности:**
- Экспоненциальная задержка
- Не повторяем при rate limit (429)
- Логирование каждой попытки
- Fallback при исчерпании попыток

### 4. Contextual Memory
**Зачем:** Чат помнит предыдущие сообщения

**Как работает:**
```
Сессия 1:
  Пользователь: "Объясни рекурсию"
  AI: [объяснение]
  
  Пользователь: "Покажи пример"
  AI: [помнит что речь о рекурсии, показывает пример]
  
  Пользователь: "А как это применить в моём коде?"
  AI: [помнит контекст, даёт конкретный совет]
```

**Оптимизация:**
- Хранение в памяти (не в БД)
- Лимит 10 сообщений
- Суммаризация старых сообщений
- Поиск по контексту

### 5. Configuration Validator
**Зачем:** Раннее обнаружение проблем с конфигурацией

**Что проверяет:**
- ✅ GROQ_API_KEY установлен
- ✅ Подключение к Groq API работает
- ✅ MCP серверы настроены (если включены)
- ✅ База данных доступна

**Вывод при старте:**
```
[Config] ✅ GROQ_API_KEY: configured
[Config] ✅ Groq API: connected
[Config] ✅ Database: connected
[Config] ⚠️  MCP enabled but no servers configured
[Config] ✅ Application ready
```

### 6. Task Classifier
**Зачем:** Точная оценка сложности заданий

**Как работает:**
```
Задание: "Напиши функцию для сортировки массива"
    ↓
AI анализирует:
  - Сложность вопроса: 6/10
  - Требуемые знания: массивы, алгоритмы
  - Время выполнения: ~15 мин
    ↓
Классификация: MEDIUM
```

**Валидация:**
- Проверка распределения: 40% easy, 40% medium, 20% hard
- Ручная корректировка при необходимости

### 7. State Machine для генерации теории
**Зачем:** Лучший контроль и обработка ошибок

**Граф состояний:**
```
START
  ↓
ANALYZE (анализ темы)
  ↓
GENERATE (генерация секций)
  ↓ (успех)        ↓ (ошибка)
VALIDATE          RETRY
  ↓ (ok)  ↓ (fail)  ↓
COMPLETE  GENERATE  GENERATE
```

**Преимущества:**
- Отслеживание прогресса
- События для UI
- Retry на уровне узлов
- Инспекция состояния

### 8. MCP Server Management UI
**Зачем:** Мониторинг и управление MCP-серверами

**Интерфейс:**
```
MCP Servers
├─ 🟢 filesystem (running)
│  └─ Tools: save_file, read_file, list_files
├─ 🟢 brave-search (running)
│  └─ Tools: search
└─ 🔴 chrome-devtools (error: connection refused)
   └─ [Retry] [View Logs]
```
