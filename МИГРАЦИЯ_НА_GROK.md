# Миграция с GigaChat на Grok

## Что изменилось

Проект теперь использует **Grok от xAI** вместо GigaChat от Сбера в качестве основного AI провайдера.

## Быстрый старт

### 1. Получите API ключ Grok

- Зайдите на https://console.x.ai
- Создайте аккаунт или войдите
- Создайте API ключ
- Скопируйте его

### 2. Обновите .env файл

Откройте файл `.env` и замените:

```env
# Старое (удалите)
GIGACHAT_AUTH_KEY=...

# Новое (добавьте)
GROK_API_KEY=ваш_ключ_здесь
```

### 3. Перезапустите проект

```bash
npm run dev
```

## Преимущества Grok

1. **Простая аутентификация** - обычный Bearer token вместо OAuth2
2. **Мощные модели** - grok-2-latest и grok-2-mini
3. **Быстрая работа** - оптимизированные модели
4. **Стриминг** - полноценная поддержка потоковой передачи
5. **OpenAI-совместимый API** - стандартный формат запросов

## Модели Grok

| Модель | Использование | Скорость | Качество |
|--------|---------------|----------|----------|
| grok-2-mini | Быстрые задачи, чат | ⚡⚡⚡ | ⭐⭐⭐ |
| grok-2-latest | Сложные задачи, JSON | ⚡⚡ | ⭐⭐⭐⭐⭐ |

Система автоматически выбирает модель в зависимости от:
- Требуется ли JSON ответ
- Размер ожидаемого ответа (maxTokens)
- Тип задачи (fast/heavy/chat)

## Fallback система

Если Grok недоступен, система автоматически переключится на:

1. **Groq** (LLaMA 3.3-70B, LLaMA 3.1-8B)
2. **Gemini** (Google Gemini 2.5 Flash Lite)

Убедитесь, что у вас настроены резервные ключи:

```env
GROQ_API_KEY=...
GEMINI_API_KEY=...
```

## Удаление старых настроек

Можете безопасно удалить из `.env`:

```env
# Больше не нужно
GIGACHAT_AUTH_KEY=...
```

## Проверка работы

После настройки запустите проект и создайте новый курс. В консоли вы увидите:

```
[AI Router] Trying grok...
[Grok] grok-2-mini - tokens: 1234 (prompt: 567, completion: 667)
[AI Router] Success with grok in 2345ms
```
